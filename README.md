# AI Workflow Setup Portfolio Project

Introduction

This repository documents the setup process for AI-assisted development and research tools, including Cursor IDE, Claude Code, Codex, and GitHub integration.

The goal of this project is to demonstrate independent problem-solving, documentation clarity, and familiarity with modern AI workflows.

-----------------------------------------------------------------------------------------------------------

Tools Installed

1. Cursor IDE
AI-powered code editor used for development and AI-assisted workflows.

2. Claude Code Extension
Installed inside Cursor for AI-assisted reasoning and coding workflows.

3. Codex Extension
Installed inside Cursor for code generation and AI-assisted development support.

4. GitHub
Used for repository hosting and version control.

-----------------------------------------------------------------------------------------------------------

Steps Completed

1. Installed Cursor IDE
2. Installed Claude Code extension
3. Installed Codex extension
4. Logged into AI tools
5. Created a public GitHub repository
6. Opened the repository in Cursor
7. Created README.md documentation
8. Initialized Git tracking
9. Committed and pushed files to GitHub

-----------------------------------------------------------------------------------------------------------

Issues Encountered and Solutions

Issue 1 : Claude Code and Codex extension did not initially appear in the Cursor Marketplace

When searching directly in the Marketplace, the extension could not be found, which initially caused confusion about the setup process.

Solution:

After further exploration, I discovered that Claude Code functionality was accessible through Cursor's integrated terminal and AI workflow tools rather than the standard extension listing. This clarified the distinction between Marketplace extensions and terminal-based AI tooling.

-----------------------------------------------------------------------------------------------------------

Issue 2 : Unable to locate the "Add File" option directly in the GitHub repository interface

While attempting to upload files through the GitHub web interface, the expected upload option did not appear.

Solution:

To resolve this, I researched alternative workflows and learned how to push files manually using Git Bash and Git commands from a local repository.

I followed GitHub and YouTube tutorials to:
- initialize the repository locally
- connect the local folder to the remote GitHub repository
- commit files
- push changes using Git commands

This helped me better understand the relationship between local repositories, Git, and remote GitHub hosting.
## Key Takeaways

This project helped me better understand AI-assisted workflows, repository management, and the importance of self-learning when working with unfamiliar tools.
